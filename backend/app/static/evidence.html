Evidence

No demos-as-proof. Controlled experiments, committed artifacts.

Two experiments back every claim on this site. Both are seeded and deterministic offline; the live run is committed to the repository down to the individual prediction. Four invariants run in CI on every push — if learning ever regresses, the build fails.

Experiment 1 · Architecture ablation

Does consolidation beat raw retrieval?

Three arms, identical tickets, identical six-memory retrieval budget per decision. The only variable is what the agent does with memory. 8 sessions × 25 tickets, seeded and noisy (category-neutral filler pollutes cosine similarity — the realistic regime).

Token economics: Mnemo reaches +10.5 pts over RAG while consuming a 50% smaller memory context per decision — 574 vs 1,155 characters. A few distilled rules carry more signal than a pile of raw transcripts. That is the practical answer to context rot.

Experiment 2 · Organization conventions — LIVE

Does memory help Qwen itself?

The fair objection to any memory agent: wouldn't a frontier model ace this zero-shot? On generic tickets — yes. So this experiment uses tickets whose correct routing depends on organization conventions no model can know a priori: refunds route to account managers by policy; "Project Falcon" tickets — invoices included — go to the white-glove technical team; Acme's sync issues are a known shipping-feed bug.

The rules Qwen wrote for itself

During consolidation, Qwen3.7-Max distilled every convention into a readable rule — rationales included, unedited output from the live run:

IF text mentions 'Project Falcon' THEN category=technical (Project Falcon hardware, workspaces, and add-ons are technical)
IF text mentions requesting or processing a refund THEN category=account (refund requests are handled under account management)
IF text contains 'Acme' OR 'warehouse feed' THEN category=shipping (Acme and warehouse integrations are tied to shipping logistics)
IF text contains 'purchase order' THEN category=billing (purchase order tracking and arrivals are managed by billing)
IF text contains 'beta' AND ('crash' OR 'break' OR 'logging out' OR 'search bar') THEN category=feedback (beta build issues are treated as product feedback)
Enforced in CI

Four invariants. Every push.

Claims decay unless something guards them. These assertions run on every commit — GitHub Actions, fresh machine, ~20 seconds:

tests/test_learning.py

Learning ordering + token economics

no-memory < episodic-RAG < Mnemo; the curve climbs; Mnemo's context is <80% of RAG's. The core thesis, as an assertion.

tests/test_org_learning.py

Conventions distill correctly

All five org conventions end up as procedural rules with the org's routing — not the surface reading.

tests/test_adaptation.py

Unlearning

After a policy flip, the refund rule ends up citing the new category, with lineage to the retired rule, within three sessions of corrections.

The product loop

Seed → fired-rule provenance → feedback actually reinforces → workspaces isolate. The console's whole story, asserted end-to-end.

Reproduce it

Two seconds. No API key. No trust.

# clone and run the full 3-arm ablation, deterministic:
git clone https://github.com/Uthmannabeel/mnemo && cd mnemo/backend
pip install -r requirements.txt
MNEMO_OFFLINE=1 python -m app.eval.harness

# the org-conventions pipeline, offline:
MNEMO_OFFLINE=1 python -m app.eval.live_harness

# with a Model Studio key, the same code paths run live Qwen3.7-Max:
python -m app.eval.live_harness --yes
results/org_experiment.json — live run backend/tests — all four suites CI runs
